## 4. 受け入れ基準（不具合対応）
- P0チケットがすべてクローズされるまで、本番切替を実施しない。
- 在庫判定の `unknown/error/out_of_stock/in_stock` が明確に区別される。
- fetch 条件や `stocking_domain` 最適化を触る変更では、alias domain を含む回帰テストが追加されている。
- DB更新ポイントが単一であることをコードレビューで確認できる。
- cron実行で失敗時に必ず非0終了となり、監視通知が飛ぶ。
